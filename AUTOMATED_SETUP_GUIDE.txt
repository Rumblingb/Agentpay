🚀 HOW TO USE
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Download the script
──────────────────────────────
Download "SETUP_PROJECT.ps1" from the files above


STEP 2: Place all your files in a "new" folder
──────────────────────────────────────────────
Your current structure:
  C:\Users\visha\agentpay\
    ├─ package.json
    ├─ tsconfig.json
    ├─ jest.config.js
    ├─ START_HERE.md
    ├─ server.ts
    ├─ logger.ts
    ├─ (all other files in root)

Create it like this:
  C:\Users\visha\agentpay\
    ├─ new\                    👈 Create this folder
    │  ├─ package.json         👈 Move all files here
    │  ├─ tsconfig.json
    │  ├─ jest.config.js
    │  ├─ START_HERE.md
    │  ├─ server.ts
    │  ├─ logger.ts
    │  ├─ payment-verification.ts
    │  ├─ init.ts
    │  ├─ (etc.)
    │
    └─ SETUP_PROJECT.ps1       👈 Put script here


STEP 3: Run the script
──────────────────────
In PowerShell (as Administrator):

```powershell
cd C:\Users\visha\agentpay
powershell -ExecutionPolicy Bypass -File SETUP_PROJECT.ps1
```

If you get an error about execution policy:

```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope CurrentUser -Force
powershell -File SETUP_PROJECT.ps1
```


STEP 4: Watch it work!
──────────────────────
The script will show:
  ✅ Creating folder structure
  ✅ Placing files in correct locations
  ✅ Renaming config files
  ✅ Verifying all files are there
  ✅ Summary showing what's complete


STEP 5: Clean up (optional)
──────────────────────────
After the script completes successfully:

```powershell
# Delete the "new" folder
Remove-Item new -Recurse -Force
```


STEP 6: Install and test
────────────────────────
```powershell
npm install
npm test
npm run dev
```

❓ TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Q: "ExecutionPolicy" error?
A: Run this first:
   Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope CurrentUser

Q: Script says files are missing?
A: Make sure ALL files were downloaded and are in the "new" folder

Q: Which merchants.ts goes where?
A: The script automatically handles this - it puts one in services/ and one in routes/

Q: Can I run the script multiple times?
A: Yes! It won't overwrite existing files, only add missing ones

Q: Do I need to rename files manually?
A: No! The script does it (_env.example → .env, etc.)
