# API Provider Partnership Outbound

## Positioning Note

AgentPay is not asking providers to become a marketplace listing.

AgentPay is offering them:

- more agent-native customers
- less setup abandonment
- secure credential handling outside agent context
- a path from one-time vaulting now to delegated auth and OAuth later

The pitch is simple:

If an agent wants a provider API, AgentPay can already route the authority, vault, funding, and execution continuity. The only missing unlock is provider-supported delegated auth, which increases completed setups and paid usage for the provider.
