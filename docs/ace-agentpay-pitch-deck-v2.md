## SLIDE 9 — AGENTPAY: THE INFRASTRUCTURE

### Ace runs on AgentPay. So will every trusted agent that comes after it.

**The four layers of AgentPay:**

| Layer | What it does |
|---|---|
| **Identity** | Every agent has a verifiable identity (`agt_*` ID, signed keys) |
| **Payment** | Agents can hold wallets, spend within policies, pay on behalf of users |
| **Trust** | Spending limits, biometric gates, approval chains — all protocol-level |
| **Execution** | Every action is recorded, recoverable, and auditable |

**The platform architecture:**

```
AgentPay Protocol
    │
    ├── Ace            (consumer travel companion — live)
    ├── Partner Agent  (enterprise travel management)
    ├── Health Agent   (appointment + prescription handling)
    └── Any agent that needs to book, pay, and be trusted
```

**Ace proves to people that agents can be trusted.**
**AgentPay proves to developers that trusted execution can be built.**
