## BLOCKER 2 — Missing mandate GET routes

`agentpay_get_mandate` and `agentpay_get_mandate_history` both 404 in production.

`aceIntents.ts` router has no `GET /:intentId` or `GET /:intentId/history`.

**Fix — add to `apps/api-edge/src/routes/aceIntents.ts`:**

```typescript
// GET /:intentId — fetch mandate record
router.get('/:intentId', async (c) => {
  const { intentId } = c.req.param();
  const sql = createDb(c.env);
  try {
    const rows = await sql<AceIntentRow[]>`
      SELECT * FROM ace_intents WHERE id = ${intentId}::uuid LIMIT 1
    `;
    if (!rows.length) return c.json({ error: 'NOT_FOUND' }, 404);
    return c.json(mapAceIntentRow(rows[0]));
  } finally {
    await sql.end();
  }
});

// GET /:intentId/history — audit timeline
router.get('/:intentId/history', async (c) => {
  const { intentId } = c.req.param();
  const sql = createDb(c.env);
  try {
    const rows = await sql<AceIntentRow[]>`
      SELECT * FROM ace_intents WHERE id = ${intentId}::uuid LIMIT 1
    `;
    if (!rows.length) return c.json({ error: 'NOT_FOUND' }, 404);
    const intent = rows[0];
    // Return synthesized timeline from status history
    return c.json({
      intentId,
      events: [
        { event: 'created', at: intent.created_at, status: 'draft' },
        intent.planned_at ? { event: 'planned', at: intent.planned_at, status: 'awaiting_approval' } : null,
        intent.approved_at ? { event: 'approved', at: intent.approved_at, status: 'approved' } : null,
        intent.executing_at ? { event: 'execution_started', at: intent.executing_at, status: 'executing' } : null,
        intent.completed_at ? { event: 'completed', at: intent.completed_at, status: 'completed' } : null,
        intent.failed_at ? { event: 'failed', at: intent.failed_at, status: 'failed' } : null,
      ].filter(Boolean),
    });
  } finally {
    await sql.end();
  }
});
```

## BLOCKER 3 — Resend email delivery not working

Registration and key-recovery emails are silently dropped.

- `POST /api/merchants/register` returns success but email never arrives
- `POST /api/merchants/recover/request` returns success but email never arrives
- No error returned to caller — caller has no idea

**Likely causes (check in order):**
1. `RESEND_API_KEY` Cloudflare Workers secret is not set on the production deployment
2. Sending domain not verified in Resend dashboard
3. Emails sending from an unverified address that Resend silently blocks

**Fix:** Verify the `RESEND_API_KEY` secret is set via `wrangler secret list --env production`,
and verify the sending domain in the Resend dashboard. Also add error logging to the
fire-and-forget email calls so failures surface in Cloudflare logs.
